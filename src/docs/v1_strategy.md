# v1 策略說明

## 目標
- 建立一個穩定的傳統機器學習基線，作為文字情感分類的起點。

## 策略
1. 採用輕量文字清洗與正規化。
2. 還原被拆開的縮寫，並保留 `num_num`、`num_extend` 這類具意義的標記。
3. 使用 TF-IDF，搭配 word n-grams `(1, 2)`，特徵上限設為 5,000。
4. 移除非情感相關的停用詞，同時保留 `not`、`no`、`but` 這類具極性判斷作用的詞。
5. 使用 Logistic Regression，並開啟 class balancing。

## 為什麼有效
- 這個版本先建立了穩定的基線，也證明在這個資料集上，精細清洗比盲目擴充特徵更重要。

## 限制
- Bag-of-words 類特徵無法完整掌握上下文語意，因此表現很快會遇到瓶頸。
