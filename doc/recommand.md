# 文本情感分類競賽解決方案分析

針對本次競賽中「半處理文本」與「語境缺失」的特性，以下提供兩套從基礎到進階的技術路徑。

---

## 方案一：傳統機器學習路徑 (Baseline)
**核心邏輯：** 利用統計學方法捕捉關鍵字詞的出現頻率，並透過整合學習模型進行分類。

### 1. 資料預處理 (Preprocessing)
由於資料中存在大量的 `num_num`、`num_extend` 以及破碎的縮寫（如 `didn t`），首先需進行標準化：
- **正規化：** 將 `didn t` 還原為 `didnot`，`i m` 還原為 `iam`。
- **標記保護：** 確保 `num_num` 標記不被視為標點符號刪除。
- **停用詞過濾：** 移除無意義的冠詞，但保留具情感色彩的否定詞（如 not, never）。

### 2. 特徵萃取：TF-IDF
使用 `TfidfVectorizer` 將文本轉為向量：
- **N-gram 策略：** 建議使用 `ngram_range=(1, 2)`，同時捕捉單個單字與雙詞組合（例如：`not good`）。
- **特徵上限：** 限制 `max_features` 以防過擬合。

### 3. 模型選擇：XGBoost / Random Forest
- **優點：** 訓練極快，對結構化特徵（詞頻矩陣）處理效果佳。
- **實作重點：** - 使用交叉驗證 (Cross-validation) 確保穩定性。
    - 調整樹的深度 (`max_depth`) 以對應模糊的分類邊界。

---

## 方案二：預訓練語言模型路徑 (Advanced)
**核心邏輯：** 利用在大規模文本上訓練過的 Transformer 模型，理解句子的深層語意與語境。

### 1. 模型選型：RoBERTa 或 DeBERTa
- **理由：** 相比於 BERT，RoBERTa 在處理長短不一的文本與非正式語言時表現更穩定，適合來源多雜的資料。

### 2. 斷詞器調整 (Tokenizer Customization)
這是此任務成功的關鍵：
- **新增特殊 Token：** 必須將 `num_num` 與 `num_extend` 加入斷詞器的字典中。
- **避免過度拆解：** 防止 Tokenizer 將 `num_num` 拆成 `n`, `u`, `m`, `_`。

### 3. 微調策略 (Fine-tuning)
- **學習率調優：** 使用極小的學習率（如 `2e-5`）進行微調，避免破壞預訓練的權重。
- **處理界線模糊：** - 使用 **Label Smoothing** 技術，應對標籤定義不夠絕對的情況。
    - 在模型最後一層加入 Dropout，提升泛化能力。

---

## 兩種方案對比分析

| 特性 | 方案一 (TF-IDF + XGBoost) | 方案二 (RoBERTa) |
| :--- | :--- | :--- |
| **開發速度** | 極快 (1-2 小時可產出結果) | 較慢 (需配置 GPU 環境與訓練) |
| **運算需求** | 一般筆電即可運行 | 需要 GPU 加速 |
| **語意理解** | 弱，僅看詞彙出現頻率 | 強，能判別反諷與上下文 |
| **應對半處理文本** | 依賴手動撰寫 RegEx 清洗 | 依賴自定義 Tokenizer |
| **建議用途** | 作為第一個提交版本確立基準分 | 爭取排行榜高分的最終手段 |

---

## 實作建議流程
1. **先做方案一：** 快速了解資料集有哪些關鍵字會導向標籤 1 或 0。
2. **分析錯誤：** 找出方案一分錯的樣本，判斷是否是因為「語氣」或「複雜句型」導致。
3. **實作方案二：** 針對方案一無法解決的模糊案例，利用 Transformer 的語意能力進行突破。
